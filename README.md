# To-Do List

список задач, JWT-авторизация, API на Nitro, данные в памяти (без БД).

## Требования

- **Node.js** 18+

В вёрстке задано **`html { font-size: 62.5% }`**, чтобы удобно считать размеры: **1rem ≈ 10px** при стандартных настройках браузера (16px).

## Установка

```bash
npm install
```

## Запуск

**Разработка** — фронт и API на одном порту `http://localhost:3000`:

```bash
npm run dev
```

**Проверить, что проект собирается:**

```bash
npm run build
```

**Просмотр собранной версии локально:**

```bash
npm run preview
```

## API

Базовый URL совпдаает с адресом приложения `http://localhost:3000`.

| Метод | Путь | Аутентификация | Описание |
|-------|------|----------------|----------|
| `POST` | `/api/auth/login` | Нет | Вход. Тело JSON: `{ "email", "password" }`. Ответ: `{ token, user }`. |
| `GET` | `/api/tasks` | Да | Список задач с пагинацией и фильтрами. |
| `POST` | `/api/tasks` | Да | Создание задачи. Тело: `{ title, description?, dueDate, isCompleted? }`. |
| `PUT` | `/api/tasks/:id` | Да | Обновление задачи. Тело: `{ title?, description?, dueDate?, isCompleted? }`. |
| `DELETE` | `/api/tasks/:id` | Да | Удаление задачи. |

### `GET /api/tasks` — query-параметры

| Параметр   | По умолчанию | Описание |
|------------|--------------|----------|
| `page`     | `1`          | Номер страницы (≥ 1). |
| `pageSize` | `10`         | Размер страницы (1–50). |
| `search`   | —            | Поиск по заголовку и описанию. |
| `filter`   | `all`        | `all` — все, `active` — активные, `completed` — выполненные. |
| `sort`     | `createdAtDesc` | `createdAtDesc` / `createdAtAsc` — по дате создания (убывание / возрастание), `status` — по статусу. |

Ответ: `{ items, total, page, pageSize }`.

### Права на задачи

Редактировать и удалять может **автор** задачи или **admin**; иначе `403`.

### Коды ошибок

- `401` — нет или неврный токен (для `/api/tasks/*`).
- `400` — неверные данные в запросе.
- `403` — нет прав.
- `404` — задача не найдена.
